Capitulo 1: Introducción - Entendiendo la web, HTML y CSS

Entendiendo la web

Concepto de webs, servidores y URL/Enlace

URLs/Enlaces: La web es como un hotel, en donde en cada habitación hay un contenido distinto. El enlace es el número de la habitación, o sea, la URL. Esa URL/Enlace se aloja en un servidor para que este disponible en internet.

Rol de los navegadores

Navegadores: Es el vehiculo entre clientes y servidores. La PC local manda una peticion, y el servidor responde enviando archivos en respuesta a esa petición (principio "solicitud-respuesta"). Ademas, los navegadores reciben y leen el codigo HTML/CSS (y demás) y lo hacen accesible al usuario.

Entendiendo HTML y CSS

¿Que es HTML?

Lenguaje de marcado: Agarra texto y lo presenta de una manera diferente. Existe HTML y Markdome (es más ligero y fácil que HTML), y se usa para hacer documentación (README.md por ejemplo). Esto significa que es un lenguaje interpretado (ejecuta el código en tiempo real)

Con HTML se arma la "ESTRUCTURA" de una página web. Cada componente de HTML son elementos (texto, imagenes, videos, forms, etc)

Cada elemento se engloba en ETIQUETAS, las cuales tienen etiquetas de apertura y cierre.

Otros temas a ver: atributos, etiquetas autocerrables, tipos de etiquetas, SEO.

¿Qué es CSS?

Las propiedades de un elemento, son declaradas, por defecto, por el navegador.

Para asignar propiedades, se inventó CSS, que es como una guía universal o standard para distintos navegadores.

CSS permite asignarle propiedades a los elementos HTML y decirle al navegador como queremos que sean mostrados los elementos.

CSS funciona mediante reglas de estilo. Estas reglas contienen selectores, bloques de declaraciones, que contienen propiedades y valores.

Tema a ver: Modos de selección de elementos, y todas las propiedades que se pueden asignar. Responsive design (adaptar la web a la pantalla).

Capitulo 2: HTML BÁSICO

Estructura de una etiqueta

Etiqueta de apertura + Contenido + Etiqueta de cierre

Todas las etiquetas tienen una función, a nivel semantico, por eso, algunas quedan obsoletas. Por ejemplo, no se recomiendan usar las etiquetas que dan "estilo" a los elementos, ya que eso se hace en CSS.

Se deben cumplir las reglas semanticas, para mejorar el SEO en los navegadores. La semantica ayuda a trabajar mejor y que cada etiqueta cumpla un rol correcto.

Las etiquetas son como bloques.

Estructura de una página web

Escribir html:5 para que VSC escriba la estructura básica de la página.

html ==> Estructura general de la página

head ==> Información y metadatos no visibles de la página. Asigna title, relaciona con otros archivos, etiquetas "meta" etc.

body ==> Parte visible de la página.

Párrafos y encabezados

Parrafos (paragraph) ==> P

Describen parrafos con contenido (texto por lo general).

Siempre deben tener una idea y un concepto en especifico.

Encabezados (headings) ==> H1 - H6

Encabezan las partes de la página web. El h1 se utiliza una sola vez por página para titular el contenido de ella. El h2 se utiliza para encabezar partes o secciones generales, y podemos seguir así hasta el h6.

Listas ==> OL/UL + LI

Definición de listas

Estan las ordenadas (orderer) y las desordenadas (unorderer). Sirven para organizar contenido o para las barras de navegación.

Al interior de los ol y ul se encuentran las li, que serían los elementos individuales de esas listas.

Listas ordenadas (orderer) ==> OL

Se enlistan con numeros o letras de forma ordenada. Es para contenido que necesita jerarquia u orden de pasos por ejemplo.

Listas desordenadas (unorderer) ==> UL

No importa el orden del contenido. Se suele usar para barras de navegacion

Enlaces o hipervinculos(básico) ==> A

Diferencia entre pagina web y sitio web

La pagina web es la codificacion individual, pero un sitio web, es un conjunto de paginas webs que conforman un proyecto, por ejemplo, una tienda online con elementos individuales que se generan en páginas webs individuales.

Estructura del enlace

La etiqueta a usar es a, y se debe usar un atributo, llamado href="". Dentro de este atributo, va la ruta del directorio local, del servidor, o una ruta externa. Se recomienda usar el "https" y no solo el "http".

Atributo target: Se usa para elegir la acción que realizará el usuario: si se usa _blank se abre en una nueva pestaña, y si se deja el que trae por defecto (_self), el enlace se abre en la misma pestaña.

Atributo title: Se usa para poner un titulo visible cuando el usuario pone el mouse sobre el elemento. Este atributo se puede usar en otros elementos tambien, sobre todo en imagenes y formularios.

Entre las etiquetas, se debe escribir el texto o boton que se le mostrará al usuario.

Imágenes ==> IMG

La etiqueta img no lleva etiqueta de cierre (self closing tag), el elemento se carga a través del atributo SRC="".

Atributo alt ==> Permite escribir un texto "alternativo" que aparecerá si la imagen no se carga correctamente.

Rutas

Definiciones

Se trata de estrategias de trabajo a la hora de confeccionar proyectos a través de diferentes directorios.

Las rutas refieren especificamente a como acceder a un directorio desde una posición (por ejemplo, desde un index). El mecanismo de acceso consiste en usar la barra / para ir a un directorio "posterior" y el comando ../ para un directorio "anterior". Estos comandos se pueden ir acumulando para entrar a un directorio dentro de otro.

En VSC, se puede poner la ruta arrastrando el directorio a un lugar del código. Tener cuidado si se cambia el nombre de las rutas relativas.

Tipos de rutas

Rutas Absolutas: son las que rutas que no importan desde donde se acceda, siempre nos dirige a un mismo lugar. EL mejor ejemplo son las URL

Rutas Relativas: Dependen del lugar de "inicio" del enlace, por lo que depende de como se escriba la ruta con / y ../, la "llamada" será la clave.

Formularios

Definiciones

Forma de pedirle datos al usuario (nombre, apellido, mail, opiniones, confirmaciones, etc).
