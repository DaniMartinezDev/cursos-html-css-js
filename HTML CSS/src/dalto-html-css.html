Capitulo 1: Introducción - Entendiendo la web, HTML y CSS

Entendiendo la web

Concepto de webs, servidores y URL/Enlace

URLs/Enlaces: La web es como un hotel, en donde en cada habitación hay un contenido distinto. El enlace es el número de la habitación, o sea, la URL. Esa URL/Enlace se aloja en un servidor para que este disponible en internet.

Rol de los navegadores

Navegadores: Es el vehiculo entre clientes y servidores. La PC local manda una peticion, y el servidor responde enviando archivos en respuesta a esa petición (principio "solicitud-respuesta"). Ademas, los navegadores reciben y leen el codigo HTML/CSS (y demás) y lo hacen accesible al usuario.

Entendiendo HTML y CSS

¿Que es HTML?

Lenguaje de marcado: Agarra texto y lo presenta de una manera diferente. Existe HTML y Markdome (es más ligero y fácil que HTML), y se usa para hacer documentación (README.md por ejemplo). Esto significa que es un lenguaje interpretado (ejecuta el código en tiempo real)

Con HTML se arma la "ESTRUCTURA" de una página web. Cada componente de HTML son elementos (texto, imagenes, videos, forms, etc)

Cada elemento se engloba en ETIQUETAS, las cuales tienen etiquetas de apertura y cierre.

Otros temas a ver: atributos, etiquetas autocerrables, tipos de etiquetas, SEO.

¿Qué es CSS?

Las propiedades de un elemento, son declaradas, por defecto, por el navegador.

Para asignar propiedades, se inventó CSS, que es como una guía universal o standard para distintos navegadores.

CSS permite asignarle propiedades a los elementos HTML y decirle al navegador como queremos que sean mostrados los elementos.

CSS funciona mediante reglas de estilo. Estas reglas contienen selectores, bloques de declaraciones, que contienen propiedades y valores.

Tema a ver: Modos de selección de elementos, y todas las propiedades que se pueden asignar. Responsive design (adaptar la web a la pantalla).

Capitulo 2: HTML BÁSICO

Estructura de una etiqueta

Etiqueta de apertura + Contenido + Etiqueta de cierre

Todas las etiquetas tienen una función, a nivel semantico, por eso, algunas quedan obsoletas. Por ejemplo, no se recomiendan usar las etiquetas que dan "estilo" a los elementos, ya que eso se hace en CSS.

Se deben cumplir las reglas semanticas, para mejorar el SEO en los navegadores. La semantica ayuda a trabajar mejor y que cada etiqueta cumpla un rol correcto.

Las etiquetas son como bloques.

Estructura de una página web

Escribir html:5 para que VSC escriba la estructura básica de la página.

html ==> Estructura general de la página

head ==> Información y metadatos no visibles de la página. Asigna title, relaciona con otros archivos, etiquetas "meta" etc.

body ==> Parte visible de la página.

Párrafos y encabezados

Parrafos (paragraph) ==> P

Describen parrafos con contenido (texto por lo general).

Siempre deben tener una idea y un concepto en especifico.

Encabezados (headings) ==> H1 - H6

Encabezan las partes de la página web. El h1 se utiliza una sola vez por página para titular el contenido de ella. El h2 se utiliza para encabezar partes o secciones generales, y podemos seguir así hasta el h6.

Listas ==> OL/UL + LI

Definición de listas

Estan las ordenadas (orderer) y las desordenadas (unorderer). Sirven para organizar contenido o para las barras de navegación.

Al interior de los ol y ul se encuentran las li, que serían los elementos individuales de esas listas.

Listas ordenadas (orderer) ==> OL

Se enlistan con numeros o letras de forma ordenada. Es para contenido que necesita jerarquia u orden de pasos por ejemplo.

Listas desordenadas (unorderer) ==> UL

No importa el orden del contenido. Se suele usar para barras de navegacion

Enlaces o hipervinculos(básico) ==> A

Diferencia entre pagina web y sitio web

La pagina web es la codificacion individual, pero un sitio web, es un conjunto de paginas webs que conforman un proyecto, por ejemplo, una tienda online con elementos individuales que se generan en páginas webs individuales.

Estructura del enlace

La etiqueta a usar es a, y se debe usar un atributo, llamado href="". Dentro de este atributo, va la ruta del directorio local, del servidor, o una ruta externa. Se recomienda usar el "https" y no solo el "http".

Atributo target: Se usa para elegir la acción que realizará el usuario: si se usa _blank se abre en una nueva pestaña, y si se deja el que trae por defecto (_self), el enlace se abre en la misma pestaña.

Atributo title: Se usa para poner un titulo visible cuando el usuario pone el mouse sobre el elemento. Este atributo se puede usar en otros elementos tambien, sobre todo en imagenes y formularios.

Entre las etiquetas, se debe escribir el texto o boton que se le mostrará al usuario.

Imágenes ==> IMG

La etiqueta img no lleva etiqueta de cierre (self closing tag), el elemento se carga a través del atributo SRC="".

Atributo alt ==> Permite escribir un texto "alternativo" que aparecerá si la imagen no se carga correctamente.

Rutas

Definiciones

Se trata de estrategias de trabajo a la hora de confeccionar proyectos a través de diferentes directorios.

Las rutas refieren especificamente a como acceder a un directorio desde una posición (por ejemplo, desde un index). El mecanismo de acceso consiste en usar la barra / para ir a un directorio "posterior" y el comando ../ para un directorio "anterior". Estos comandos se pueden ir acumulando para entrar a un directorio dentro de otro.

En VSC, se puede poner la ruta arrastrando el directorio a un lugar del código. Tener cuidado si se cambia el nombre de las rutas relativas.

Tipos de rutas

Rutas Absolutas: son las que rutas que no importan desde donde se acceda, siempre nos dirige a un mismo lugar. EL mejor ejemplo son las URL

Rutas Relativas: Dependen del lugar de "inicio" del enlace, por lo que depende de como se escriba la ruta con / y ../, la "llamada" será la clave.

Formularios

Definiciones

Forma de pedirle datos al usuario (nombre, apellido, mail, opiniones, confirmaciones, etc).

Le da un formato a los datos ingresados, y facilitan la interfaz del usuario

¿Cómo se compone un formulario?

Primero se coloca la etiqueta form para dar el elemento de bloque que contendrá el formulario.

Luego, viene la etiqueta input, que indica al usuario que deberá ingresar un dato. Dentro del input, se colocan los atributos que definiran el tipo de dato, y demás funcionalidades.

Atributos posibles para input:

  • Type ==> Indica el tipo de dato a ingresar. Es el más importante, y hay varios tipos posibles. Los más destacados son:
    • Text==> Ingreso de texto
    • Color==> Seleccionar un color
    • Email ==> Es como un texto, pero requiere el formato de un mail (arroba con texto atras).
    • PasswordNo se muestra el contenido escrito.
    • CheckboxGenera una caja para dar okey
    • RadioPermite generar una de entre varias opciones.
    • FilePermite seleccionar y subir un archivo.
    • DatePermite seleccionar una fecha.
    • SummitGenera el boton de "Enviar" y enviar los datos al servidor. Con el atributo value se le puede cambiar el texto a mostrar.
  • Required==> Obliga al usuario a completar el input con algun dato. Es un atributo de funcionalidad, no visual.
  • Name==> Sirve como un identificador para el input, que servirá a la hora de ser usado por el servidor. Cuando esté lo reciba, verificará el dato ingreso y actuará en consecuencia. Es un atributo de funcionalidad, no visual.
  • Minlength & Maxlength = "..."==> Establece un minimo y/o maximo de datos a ingresar.
  • Placeholder==> Muestra un texto o dato de ejemplo, para guiar al usuario. Es un atributo visual, no funcional.
  • Readonly==> Hace que el input solo este disponible para su lectura.
  • Disabled==> Hace que el input no pueda ser modificado ni seleccionado. Sobre todo, no se enviará la información al servidor. Podría usarse para temrinos y condiciones que deban ser leidas y no enviadas al servidor.
  • Value==> Es el dato que recibirá el servidor, ya que lo va a almacenar en ese "Value". Se usa con JavaScript.
  • Method==> Establece la manera en la cual se enviaran los datos al servidor. Se usa con JavaScript.

Capitulo 3: CSS BÁSICO

Introducción a CSS

Definiciones

La Cascada se refiere a como se van definiendo propiedades a los elementos de HTML, y como van "cayendo" según el orden de selección.

Se trata de entender como funciona cada propiedad y como se van mezclando y relacionan esas propiedades, ya demás, como se relacionan los elementos entre sí (por ejemplo, en la herencia de elementos).

Integración de CSS a los elementos HTML

Primer método ==> Estilos en línea: Se le da style="..." como si fuera un atributo del elemento afectado. Se pueden dar varios estilos a la vez. Pero no esta recomendado.

Segundo método ==> Etiqueta Style dentro de HTML. No se recomienda tampoco.

Tercer método y el más recomendado ==> Linkear una hoja de estilo (en el head del HTML) a una hoja de estilo. Se usa la etiqueta para vincular archivos.

Selectores

Formas de seleccionar elementos

Selección por elemento ==> Indicar el tipo de elemento, seleccionando así todos los elementos que sean del mismo tipo, por ejemplo, párrafos. Se suele usar para los body o elementos "raiz", pero por lo general, se suele usar la selección por clase.

Selección por clase ==> Es la manera recomendada y más utilizada. Se trata de darle clases a los elementos HTML de manera individual o grupal, a traves de la etiqueta class como si fuera un atributo más del elemento HTML a seleccionar. Se debe indicar con un punto al inicio de la regla de declaración en el CSS. Se pueden "anidar" varias selecciones de clases, poniendo un punto en el medio de cada clase dentro de la familia de elementos HTML. A su vez, se le pueden dar varias clases a un mismo elemento, separando por un espacio entre las clases.

Selección por identificador único (id) ==> Se usa para un único elemento, no se puede repetir. Se debe usar el comando # para seleccionar.

Propiedades de fuente y texto

Propiedades más utilizadas

color ==> Color del texto o elemento.

font-family ==> Tipografía a utilizar. Se pueden colocar varias fuentes, usando una coma entre ellas, a modo de "fallback".

font-size ==> Tamaño de letra. Se suele usar px, em o rem para medir el tamaño de la fuente.

font-weight ==> Indica el grosor de la fuente, se usa el valor "bold" para resaltar como "negrita" o valores númericos de entre 100 a 900, siendo el 100 el menos grueso. Estos números va a depender de la fuente, ya que muchas solo tienen dos o tres opciones definidas (400 o 700 por ejemplo). Se usa el esquema de rangos numericos, y se elige el número más cercano para asignar la propiedad, por ejemplo, el bold se suele usar para el 700. El valor "bolder" elige el "siguiente escalón" del grosor que se viene utilizando.

font-style ==> Da inclinación al texto. El valor normal es el que viene por defecto. El valor italic inclina la letra y el valor oblique permite manejar mejor la inclinación asignando un valor de "numero + deg" (40deg = 40 grados por ejemplo).

text-align ==> Alinea el texto dentro de su contenedor. El valor left inicia el texto a la izquierda, el valor right inicia el texto a la derecha, usado en lenguas arabicas. El valor center mueve el texto al centro. Por último, el valor justify funciona como el justificado de Word, aumentando el tamaño entre letras.

text-decoration ===> Le da una linea arriba con el valor overline o abajo con underline (es decir, subrayado), o puede tachar el texto con el valor line-through. Se puden mezclar usando dos o tres. Las dos últimas son las más usadas. El valor por defecto es none para la mayoria de elementos, excepto para los links. Se puede sacar el subrayado por defecto para los links, para personalizar navegadores o botones con enlaces.

line-height ==> Asigna la altura entre lineas. Funciona como el Word, con valores númericos. Se suele usar el valor 1.3 o 1.4 por lo general.

letter-spacing ==> Asigna distancia entre letras. Se suele usar en botones por lo general.

text-transform ==> Cambia el tipo de letra entgre mayuscula y minuscula. El valor uppercase convierte a todas las letras en mayuscula. El valor lowercase convierte todo a minuscula, el valor capitalize pone la primera letra en mayuscula.

text-shadow ==> Da sombra al texto.

word-wrap ==> Con el valor break-word corta la oración en silabas para dar el salto de línea.

word-break ==> Con el valor break-all corta la palabra por donde termina la línea, sin importar la silaba.

Tipografias externas

Se puede importar familias de fuentes a través de dos mecanismos, uno más facil y otra más complicada.

Primera opción ==> Google Fonts: Elegir las fuentes y linkear el enlace que brinda la pagina, en el head del HTML.

Segunda opción ==> Instalación de fuentes en una carpeta por separado. Se deben descargar las fuentes y copiarlas en el proyecto. A su vez, se crea una hoja de estilo exclusivo para las fuentes, y se configura cada fuente en detalle, con el comando @font-face y desagregando el tamaño de fuente.

Box model (Modelo de la caja)

Definiciones

Concepto que describe como se diseñan y colocan los elementos en una página web. En HTML todo se considera una caja, y estas cajas van cambiando de posición, tamaño, muestra o no, y demás propiedades.

}

En resumen, el box model se compone de cuatro elementos:

  1. Content o Contenido ==> Es la zona donde se ubica unicamente el contenido propiamente dicho.
  2. Padding a Relleno ==> Es la zona que recubre al contenido y esta antes que el borde de la caja.
  3. Border o Borde ==> Es la zona que delimita a la caja, y puede mostrarse o no, pero siempre esta presente (salvo que se indique que su tamaño es 0).
  4. Margin o Margen ==> Si bien no forma parte de la caja, es la zona que rodea a esta y la separa de las otras cajas (es decir, otros elementos.)

Importante ==> El Content, Padding y el Border forman parte de la caja, mientras que el Margin no. Por ello, el Margin solo afecta la distancia, y no los atributos "visuales" en sí (como el color o presentación).

Width & Height (Ancho y Alto)

Estas dos propiedades le asignan el ancho y el alto del elemento seleccionado. Importante ==> Cuando se usan porcentajes para dar valor a las cajas, ese porcentaje será en relación a la caja "contenedora", es decir, a la caja "padre". Esto no aplica cuando se usan "px" o "viewport".

Una vez asignadas las propiedades, se le pueden brindar las demás propiedades para manipular la "caja".

Shorthands (Atajos de escritura)

Concepto de "Shorthand" ==> Es una manera "acortada" y simplificada de escribir las propiedades y valores de las declaraciones.

Existen varios tipos de Shorthand. Por ejemplo, se puede reemplazar el "top" (arriba), "right" (derecha), "bottom" (abajo) y "left" (izquierda) escribiendo estos cuatro valores en una sola línea. Esto se hace declarando estos cuatro valores en el sentido horario del reloj ==> (valor 1 = top (arriba); valor 2 = right; valor 3 = bottom y valor 4 = left). Si solo se quiere diferenciar por el par ancho y alto, se escriben solo dos valores, donde el primero es el Height (Alto) y el segundo valor es el Width (Ancho). Por último, si el valor será el mismo para los cuatro lados, entonces sol se debe escribir un valor.

Cuando un valor a dar es cero, no requiere escribir la unidad de medida (o sea, solo escribir el cero, sin px o rem o vw).

Padding & Margin (Relleno y Margen)

Padding ==> Es el espacio entre el contenido y el borde de una caja. Aumenta el tamaño que hay entre estos dos atributos.

Margin ==> Permite separar los elementos (cajas) entre sí.

Efecto "margin colapse" ==> hace que el margen "mayor" predomine entre los margenes de cada caja. De este modo los margenes se colapsan y se evita "sumar" los margenes diferentes. Cuando los margenes son iguales, tampoco se suman, sino que queda el valor "igual". Esto pasa con los elementos "hermanos", o sea, los elementos que estan dentro de la misma caja "padre" que los contiene. Este efecto se puede alterar con CSS Flexbox y CSS Grid.

Border (Bordes)

Cuatro propiedades modificables:

  • border-widht ==> Modifica el tamaño, generalmente se suele usar px.
  • border-style ==> Modifica el estilo. Estan disponibles cuatro estilos, aunque el que más se usa es "solid". Los otros son dashed" (intermitente), el "groove" (3D), el "dotted" (punteado" y por último, el valor "hidden" que oculta el borde, pero sigue existiendo.
  • border-color ==> Modifica el color. Si se pone el valor "transparent" va a afectar al tamaño de la caja, a diferencia de poner "hidden" en el border-style.
  • border-radius ==> Modifica el radio de las esquinas. Se usa de dos modos, con px (redondeo más leve) o con porcentajes (hace un redondeo más grande, ya que afecta desde el centro del elemento). Si se pone un 100% a un elemento cuadro, se convierte en un circulo.

Importante ==> Las primeras tres propiedades se usan en conjunto, a través del Shorthand "widht-style-color". El border-radius no es obligatorio, y se debe declarar en una línea aparte.

Modificación de los lados del borde ==> Usar el atributo top/right/bottom/left para modificar un lado en concreto. Por defecto, se modifican los cuatro lados de los bordes.

Eliminación de bordes de un elemento ==> Se pone border = none para sacarle un borde a un input por ejemplo.

Box Sizing (Tamaño de la caja)

Es la propiedad de CSS que calcula el total del tamaño de la caja. Existen dos formas de calcular:

  • Content-box ==> Es el predeterminado, los valores "weitd" y "heigth" afectan al Content, y entonces, las demás "capas" (Padding & Border) se le suman a ese valor determinado, haciendo incrementar a la "caja" aún más del valor del content. No es el método recomendado en la actualidad, porque suele "deformar" los tamaños y es dificil de calcular medidas. Se declara con box-sizing = content-box, aunque como viene por "defecto", no suele usarse.
  • Border-box ==> Se posiciona el tamaño desde el Border, por lo tanto, la caja va a tener una medida fija, y lo que se va ajustando es el Content. Permite un mayor control de las cajas. Se declara con box-sizing = border-box.

Colores en CSS

Formas de dar color en CSS

  • RGB/RGBA ==> Las siglas significan Red Green Blue, y son los tres colores primarios de la luz. Cada uno de ellos contiene 256 valores, que van del 0 al 255 (el más alto). El Alpha controla la opacidad de esas transparencias (por eso es RGBA). Se define de la siguiente manera color = rgba(255, 255, 255, 50). Hay páginas como RGBA Picker para poder calcular el color a elegir.
  • HSL/HSLA ==> No se suele utilizar, se basa en un circulo de colores de 180 grados, la saturación y la luminancia.
  • Hexadecimal ==> Permite la misma cantidad de colores que RGB. El sistema se basa en un codigo de 6 caracteres, divididos en tres pares (como el RGB) en donde el primero, el tercero y el quinto van a definir la escala de RGB, y el segundo, cuarto y sexto van a definir al caracter anterior, para brindar más especificidad. Los caracterres tienen 16 posibles valores, que van desde el "0" (el más bajo) hasta la letra "f" (el más alto). Se pueden usar abreviaturas en el código, cuando se repitan caracteres, pasando de un #ffffff a un #fff por ejemplo. Al igual que antes, agregando uno o dos caracteres más, se puede definir la opacidad, donde el "0" da transparencia total, y la "f" una opacidad total.

CSS Units (Unidades de medidas)

Definición:

Se suele dividir en dos grandes grupos de unidades:

  • Unidades Absolutas: Son unidades que "no varian" o no tienen relación directa con otros elementos. Sirve sobre todo para tener un control de las cajas, pero como punto negativo, es malo en temas de "design responsive". Se incluye en este grupo a:
    • Pixeles (parte más minima de luz del monitor). Se usa normalmente para definir el alto y ancho de imagenes, las tipografias, y los Padding, Border y Margin.
    • Centimentos.
    • Milimetros.
    • Pulgadas.
  • Unidades Relativas: Son unidades que dependen de otro valor asociado. Son muy usadas para el "design responsive". Tienen una relación directa con el tamaño de la ventana (viewport) o con el elemento contenedor (em) o el elemento raiz (rem) del elemento que se está modificando. Se incluyen:
    • Porcentajes (%) que estan en relación a la caja contenedora (no se usa para textos)
    • "em": vale el font-size del elemento "padre" o "abuelo" (si el "padre" no esta definido) y lo multiplica por ese valor del "padre" (3em multipica por 3, pasando de 10px a 30 px por ejemplo), sirviendo para dar coherencia a las fuentes. Por eso, se recomienda su uso en tipografias.
    • "rem" hace lo mismo que el "em", pero va a buscar al elemento "raiz" del elemento html.
    • Valores viewport: se usan para medir en relación a la ventana activa, usando el vw para medir el alto y el vh para el alto disponible, este último es muy útil para ventanas modales o para diseños muy responsivos.

Background (Fondos)

Propiedades:

background-color ==> Le da color al fondo del elemento. Con el último decimal del hexadecimal se le puede dar más o menos transparencia.

background-image: url("") ==> La URL enlaza una imagen que puede usarse como fondo de un elemento (o sea, no es una etiqueta "img") y a partir se ahi se le puede dar más propiedades dentro de la "caja" con diferentes "background".

background-size ==> Se usan porcentajes por lo general, entonces adapta la imagen al contenedor. Con un 100% adapta la imagen para que ocupe todo el ancho del contenedor. Con el valor contain obliga a la imagen a adaptarse y que entré al menos una vez en el contenedor. Por otro lado, el valor cover hace que la imagen se adapte y ajuste a la resolución del contenedor.

background-position ==> Se usa para acomodar la imagen en el contenedor de manera más controlada, con los valores top, bottom, right, center y left. Tambien se pueden usar pixeles determinados para posicionar la imagen al contenedor.

background-repeat: no-repeat ==> Hace que la imagen no se repita, cambiando el valor por defecto "repeat". A los costados de la imagen, quedaría con el background del elemento.

background-attachment: fixed ==> Permite mostrar la imagen con el scroll, creando un efecto de deslizamiento. El que viene por defecto es scroll.

Shorthand de background

background ==> El orden para colocar los parametros es: (1) url (imagen), (2) position, (3)size, (4)repeat o no-repeat, (5)attachment y (6)color.

Gradients (Gradientes)

Definiciones

Es un tipo de fondo o transicion de colores, es decir, una variación matematica entre dos colores. Se usan para tener variaciones de tonalidades, por lo general, de una misma gama, tendiendo a lo más oscuro, claro o transparente.

Se aplican con la propiedad background o background-color.

Tipos de gradientes

Gradientes lineales: Se inician con la función linear-gradient(). Esta función pide dos parametros: El primer y el segundo color (o una transparencia), separado por coma. Para cambiar la dirección, se debe agregar en el código el comando to left / to right / to top / to bottom). A su vez, se pueden dar porcentajes de uso de cada uno de los colores, para que la graduación no ocupe todo el contenedor. Por ejemplo, un código sería background: linear-gradient(to right, transparent 30%, green).

Gradientes circulares: Se inician con la función radial-gradient(). Por lo general tiene poco uso. El gradiente se genera hacia el centro del contenedor.

Gradiente conico: Se inician con conic-gradient().

Shadows (Sombras)

Definiciones

Las sombras son importantes porque agregan realismo, agregan jerarquia visual a algun elemento, mejoran la legibilidad. No hay que confundir la sombra con el borde.

Tipos de sombras

box-shadow (Sombra a la caja): Los parametros que se deben escribir son: (1) cantidad de sombra a la derecha (a la izquierda se va con porcentaje negativo),(2) cantidad de sombra hacia abajo (hacia arriba se va con porcentaje negativo), (3) cantidad de desenfoque o difuminado de la sombra, (4) cantidad de expansión de la sombra y (5) color de la sombra. El parametro más usado es el desenfoque y el color (aunque el más usado suele ser el negro, con una opacidad baja).

text-shadow (Sombra al texto): Se repite los parametros de arriba.

filter: drop-shadow (Sombra a imagenes PNG): Permite que la sombra inicie en el centro de la imagen, para utilizar el canal Alpha (transparencia) de la imagen, y de este modo, la sombra no inicia desde los bordes, como con box-shadow.

Capitulo 4: HTML AVANZADO

Metatags, comments & favicons (Metaetiquetas, comentarios e iconos)

Definiciones

  • Comentarios ==> Es un codigo legible solo para el programador, para señalizar el inicio de una sección o para dejar un mensaje a futuros desarroladores del código.
  • Favicons ==> Es el icono que figura en la pestaña del navegador. Se escribe con la etiqueta . Se recomienda una imagen de 32x32px, o de 64x64px.
  • Metatags ==> Son etiquetas quie tienen la unica finalidad de escribir ciertos aspectos de la página web. No son visibles para el usuario promedio. SOn importantes para el SEO, para la vista previa en redes sociales, etc. Los ejemplos más usuales son:
    • meta charset="UTF-8" ==> Sirve para evitar errores de lectura de caracteres especiales.
    • meta name="viewport" content="width=device-width, initial-scale=1.0" ==> El "width=device-width" sirve para que la página escale en todos los dispositivos. El "initial-scale" se encarga del zoom inicial de una página.
    • El atributo "name" refiere a lo que se esta declarando y el atributo "content" refiere al contenido a declarar.
    • meta name="description" ==> Se usa para describir la página, y es la leyenda que se previsualiza en los buscadores. Se recomiendan unos 100 caracteres.
    • meta name="robots" content="" ==> Se usa para que los motores de búsqueda sepan que hacer cuando encuentran la página. Si se pone "no-follow" no se indexará, si se pone "nosnippet" no se permitirá usar información de la página en vista previa. El "noarchive" prohibe que el buscador guarde la página en el cache, se suele usar para páginas con muchos cambios frecuentes. El "noimageindex" impide indexar imagenes internas. No se recomienda por cuestiones de SEO.
    • meta name="OG:..." ==> Las metatags de Open Graph sirven para personalizar las vistas previas de la página en redes sociales. El mejor sitio para previsualizar tarjetas en redes es Metatags.io. Se encuentran en este grupo de metatags las siguientes opciones:
      • "og:title" ==> Le da el titulo principal.
      • "og:image" ==> Brinda una imagen predefinida.
      • "og:description" ==> Da una breve descripción.
      • "og:url" ==> Indica la URL a donde se lo debe redirigir al usuario.

Textarea

Definiciones

Se parece a un input, pero contiene caracteristicas propias. Permite crear areas de texto más grandes que un input "normal".

No se debe contener dentro de un formulario, como con el input.

Indicar en CSS el font-family en especifico, porque ya trae el suyo.

Propiedades CSS aplicables:

  • resize ==> permite configurar la posibilidad de permitirle al usuario ampliar (o no) el area de texto. Los valores son: (1) both (por defecto, permite al usuario aumentar el tamaño en pantalla), (2) vertical (permite el desplazamiento vertical unicamente), (3) none (impide la modificación del tamaño).
  • min-width / max-width ==> configura el ancho minimo o maximo del textarea.
  • min-width / max-heigth ==> configura el alto minimo o maximo del textarea. Se recomienda usar ambos a la vez.
  • form-sizing: content ==> El textarea se adapta al contenido y no se precisa JavaScript.

Atributos HTML aplicables:

  • Maxlength ==> Establece la cantidad maxima de caracteres a escribir.
  • Rows ==> Establece la cantidad maxima de lineas a escribir.
  • Placeholder==> Muestra un texto de ejemplo para guiar al usuario. Es un atributo visual, no funcional.
  • Required==> Obliga al usuario a completar el input con algun dato. Es un atributo de funcionalidad, no visual.
  • Readonly ==> Es un atributo que hace que el texarea solamente este disponible para lectura. Tambien se usa en inputs.
  • Disabled==> Hace que el input no pueda ser modificado ni seleccionado. Sobre todo, no se enviará la información al servidor. Podría usarse para temrinos y condiciones que deban ser leidas y no enviadas al servidor.

Labels

Definiciones

Es una forma de etiquetar mejor un input, mejorando la accesibilidad. Hace que todo lo contenido en el input, al presionarlo, se posicione el cursor dentro del input.

Para eso, se debe poner antes del input el comando label. A continuación, se debe póner el titulo del input, que guiará al usuario.

Select, Datalist & Option

Definiciones

Son una especie de input, en donde el usuario deberá elegir alguna opción. Los Select y Datalist son dos tipos diferentes, que en su interior contienen Options. Parecido a los "ul" y "ol" con las "li".

Select ==> Son un tipo de campo de entrada que permite seleccionar una opcion de entre varias opciones. No se le puede personalizar el Padding o Margin, solamente el color de fondo o de texto. Se necesitan frameworks para ello.

Datalist ==> Son un tipo de campo de entrada con datos precargados, que a la medida que el usuario escribe, se facilita la elección del dato. En el atributo "Value" se debe indicar la opción a elegir. Si se quiere poner una descripción al elemento de lista, se lo puede poner entre los simbolos de cierre del datalist.

Parametros comunes

Ambos usan Option para poder elegir los datos. Se usan "name" e "id" para especificar el uso. El atributo "value" en los Options va a facilitar la carga de datos en el servidor, evitando acentos o caracteres especiales.
